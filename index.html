HTML

  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • As developers, one of the first things we must think about when developing a webpage is building a sturdy "frame" using HTML.
  • The head element contains information about the webpage, or metadata,that helps the browser to render the page correctly. Elements included in the header are not visible when a user looks at a webpage.
  • The body element represents the visible content shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • The body and head elements are examples of a semantic HTML element. In semantic HTML elements, the name of the element describes the content and the role it plays on a webpage.
  • Not all elements are semantic, though. It is still common to see non-semantic elements such as div to create a division in the page, or span to create an inline division. Sometimes these are simply the most efficient solution to a structural problem the developer is trying to solve.
  • The most important thing to keep in mind is that it is best practice to use semantic elements whenever possible, especially when the content is visible to users.

Git

Steps to opening a GitHub repo in local environment using CLI

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: checkout the main branch
  • git pull origin main: pull the main repo into local environment

Steps to save our work to GitHub using CLI

  • git status: Make sure on the correct branch from command line
  • git add -A: Command to commit all files in the working branch
  • git commit -m "added code...": Commit to repository with the message flag, look for changes in the output
  • git pull origin main: Sync our local branch with the remoet branch in GitHub, good habit to always pull the base branch into your feature branc before pushing code, look for already up to date
  • git push origin 'branch-name': without quotes, Pushes all changes to the main default branch
  • Navigate to GitHub and the pushed branch. Click Pull Requests.
  • Create Pull Request and add details.
  • Merge pull request and confirm pull request.
